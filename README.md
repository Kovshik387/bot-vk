# VK-бот навигации пациентов по ДМС

Прототип разработан для темы: «Разработка информационной подсистемы навигации пациентов в рамках системы ДМС».

Бот размещается на стороне страховой компании и выполняет организационную навигацию пациента:

- проверяет наличие пациента в системе ДМС;
- проверяет действующий статус полиса;
- показывает доступные медицинские организации;
- разделяет услуги на прямой доступ и услуги с согласованием;
- формирует заявки на запись, сдачу анализов и согласование;
- передает нестандартные обращения администратору;
- показывает статус последних заявок.

Бот не ставит диагнозы, не назначает лечение и не хранит медицинскую карту пациента.

## Состав проекта

- `bot.py` - основная логика VK-бота и сценарии диалога.
- `storage.py` - SQLite-хранилище, демо-пациенты, услуги, клиники и заявки.
- `keyboards.py` - клавиатуры ВК.
- `config.py` - чтение настроек из `.env` или переменных окружения.
- `.env.example` - пример файла настроек.
- `requirements.txt` - зависимости Python.

## Запуск

1. Создайте и активируйте виртуальное окружение:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

2. Установите зависимости:

```powershell
pip install -r requirements.txt
```

3. Скопируйте `.env.example` в `.env` и заполните значения:

```powershell
Copy-Item .env.example .env
```

В `.env` нужно указать:

```env
VK_TOKEN=токен_сообщества_вк
GROUP_ID=id_сообщества
DB_PATH=dms_navigation.db
ADMIN_IDS=123456,789012
```

`ADMIN_IDS` можно оставить пустым. Если указать ID администраторов через запятую, бот будет отправлять им новые заявки.

4. Запустите бота:

```powershell
python bot.py
```

## Настройка сообщества ВК

В сообществе ВК нужно включить сообщения сообщества и выдать токен с доступом к сообщениям. Бот использует Long Poll API через библиотеку `vk_api`, поэтому отдельный веб-сервер для прототипа не требуется.

## Django-админка

Для редактирования данных через сайт добавлена простая Django-админка поверх той же базы `dms_navigation.db`.

Запуск локально:

```powershell
python manage.py migrate
python manage.py bootstrap_admin --username admin --password admin12345
python manage.py runserver 127.0.0.1:8000
```

После запуска откройте:

```text
http://127.0.0.1:8000/
```

Локальный логин:

```text
Логин: admin
Пароль: admin12345
```

Через админку можно изменять:

- пациентов и активность полиса;
- программы и номера ДМС;
- услуги, категории и необходимость согласования;
- медицинские организации;
- связь услуг с медицинскими организациями;
- заявки пациентов и их статусы;
- конкретную инструкцию пациенту по каждой заявке;
- привязки VK ID к пациентам.

Пример инструкции после обработки заявки:

```text
Заявка одобрена. Позвоните в Медцентр Здоровье по телефону +7 (473) 200-10-10
и подтвердите время приема. При себе нужно иметь паспорт и полис ДМС.
```

Перед публикацией на сервере поменяйте `DJANGO_SECRET_KEY`, отключите `DJANGO_DEBUG`, задайте `DJANGO_ALLOWED_HOSTS` и смените пароль администратора.
